🎮 前端重新編譯指引

Cocos Creator 專案編譯步驟

✅ 準備工作已完成

  • Cocos Creator 已關閉
  • 快取目錄已清除
  • 前端代碼已修改（統一使用 Protobuf）
  • 後端已實現（完整 Protobuf 支援）

開啟 Cocos Creator

方法 A: 從桌面或開始選單

尋找並雙擊 "Cocos Creator" 圖示

方法 B: 從專案資料夾

1. 開啟檔案總管
2. 導航到: C:\projects\game152Dev\pss-on-00152
3. 找到專案檔案並開啟

方法 C: 從 Cocos Dashboard

開啟 Cocos Dashboard 並選擇 "pss-on-00152" 專案

等待專案載入

⏳ 重要！請耐心等待 5-10 分鐘

載入過程中會看到：

  • 📊 右下角顯示進度條
  • 📝 Console 輸出載入訊息
  • 🔄 "Compiling..." 或 "Indexing..." 文字
等到以下狀態才算完成：
✅ 右下角進度條消失
✅ Assets 面板顯示所有資源
✅ Console 沒有紅色錯誤
✅ 可以點擊 Scene 中的物件

開啟 Build 視窗

方法 A: 使用選單

點擊頂部選單: Project → Build...

方法 B: 使用快捷鍵

按下: Ctrl + Shift + B

設定編譯選項

1. Platform (平台)

選擇: Web Desktop

2. Build Path (輸出路徑)

保持預設值 (build/web-desktop)

3. Debug (調試模式)

取消勾選（正式版執行較快）

開始編譯

點擊視窗底部的 "Build" 按鈕

⏳ 預計編譯時間: 3-5 分鐘

編譯過程中 Console 會顯示：

Building... Compiling scripts... Generating assets... Packing... Build succeeded!

驗證編譯成功

✅ 編譯成功的標誌：
• Build 視窗顯示 "Build succeeded!"
• build/web-desktop 目錄有新檔案
• Console 無紅色錯誤訊息

🧪 測試編譯結果

1. 啟動 Spin Server cd C:\projects\game152Dev\gameServer python spin_server.py
2. 開啟遊戲

🎮 開啟遊戲測試

3. 檢查 Console 日誌

✅ 正確的日誌：

[DEBUG] bksend - sending binary data, byteLength: 29 [DEBUG] LoginRecall (LocalServer) status: kSuccess [DEBUG] StateRecall - status_code: kSuccess ✅ Reel 初始化成功！

❌ 如果看到這個就是錯的：

[DEBUG] bksend - sending JSON string, length: 97
如果還是看到 JSON，請：
1. 確認編譯已完成
2. 清除瀏覽器快取 (Ctrl + Shift + Delete)
3. 強制重新整理 (Ctrl + F5)
